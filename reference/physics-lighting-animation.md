# Cluster 5 — 2D Physics, 2D Lighting (URP), Sprite Shape, 2D Animation

Facts extracted from Unity 6 (6000.0) Manual + package docs. `[tags]`: concept | default | api | gotcha | perf | workflow.

## Rigidbody 2D
- **[concept]** Rigidbody 2D enables physics-based behavior (gravity, mass, drag, momentum). Colliders attached to bodies are what actually collide; Rigidbody 2Ds cannot collide without Collider 2Ds attached — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/rigidbody/body-types/dynamic/dynamic-body-type-reference.html
- **[concept]** Body Type enum: **Dynamic** (default), **Kinematic**, **Static** — sets movement behavior and how it interacts with Collider 2Ds. Dynamic: moves under simulation, finite mass/drag, affected by gravity+forces, collides with every body type. Kinematic: moves under explicit control, not affected by forces/gravity. Static: does not move under simulation — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/rigidbody/body-types/dynamic/dynamic-body-type-reference.html
- **[gotcha]** Do NOT use the Transform to set position/rotation of a **Dynamic** Rigidbody 2D — the simulation repositions it by velocity. Move it via forces/velocity instead — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/rigidbody/body-types/dynamic/dynamic-body-type-reference.html
- **[concept]** **Material**: Physics Material 2D applied to all attached Collider 2Ds. Default `None` uses the Physics 2D window default. A Collider 2D's own Material property overrides the Rigidbody's — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/rigidbody/body-types/dynamic/dynamic-body-type-reference.html
- **[default]** **Simulated**: enabled by default. When disabled, the Rigidbody 2D and its attached Collider 2Ds and Joint 2Ds do not participate in the physics simulation — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/rigidbody/body-types/dynamic/dynamic-body-type-reference.html
- **[concept]** **Use Auto Mass** (default disabled): auto-computes mass from attached Collider 2D. When enabled, the **Mass** field is greyed out — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/rigidbody/body-types/dynamic/dynamic-body-type-reference.html
- **[api]** Fields renamed in Unity 6: **Linear Damping** (formerly Linear Drag) and **Angular Damping** (formerly Angular Drag) — drag coefficients for positional / rotational movement — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/rigidbody/body-types/dynamic/dynamic-body-type-reference.html
- **[default]** **Gravity Scale**: degree the body is affected by gravity. Default value is **1** (0 = ignore gravity) — src: https://docs.unity3d.com/6000.0/Documentation/ScriptReference/Rigidbody2D-gravityScale.html
- **[concept]** **Collision Detection** enum: **Discrete** (default; fast movers may overlap/pass through) and **Continuous** (prevents tunnelling; higher CPU cost) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/rigidbody/body-types/dynamic/dynamic-body-type-reference.html
- **[concept]** **Sleeping Mode** enum: **Never Sleep** (disables sleeping — avoid where possible), **Start Awake** (default), **Start Asleep** (starts asleep, wakes on collision) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/rigidbody/body-types/dynamic/dynamic-body-type-reference.html
- **[concept]** **Interpolate** enum: **None** (default), **Interpolate** (smooth from previous frame position), **Extrapolate** (smooth from estimated next-frame position) — reduces visual jitter of physics motion at runtime — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/rigidbody/body-types/dynamic/dynamic-body-type-reference.html
- **[concept]** **Constraints**: Freeze Position (X and Y world axes, independently selectable) and Freeze Rotation (Z world axis) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/rigidbody/body-types/dynamic/dynamic-body-type-reference.html
- **[concept]** **Layer Overrides**: Include Layers, Exclude Layers (override the global Layer Collision Matrix per-body) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/rigidbody/body-types/dynamic/dynamic-body-type-reference.html
- **[api]** Scripting: `Rigidbody2D.linearVelocity`/`angularVelocity` (velocity control), `Rigidbody2D.position`/`rotation` (pose). `linearVelocity` replaces the old `velocity` in Unity 6 — src: https://docs.unity3d.com/6000.0/Documentation/ScriptReference/Rigidbody2D.html

## Collider 2D (types + shared fields)
- **[concept]** 2D collider types: **Box**, **Circle**, **Capsule**, **Polygon**, **Edge**, **Composite**, **Custom**, **Tilemap** Collider 2D. All 2D collider names end in "2D". You cannot mix 3D GameObjects with 2D colliders or vice versa — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/collider-2d-landing.html
- **[concept]** Box Collider 2D = rectangle/square (axis-aligned, edges parallel to x/y); Circle = circular; Polygon = freeform solid; Edge = freeform/open path; Capsule = pill shape — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/collider-2d-landing.html
- **[concept]** Shared Collider 2D fields: **Material** (Physics Material 2D — friction/bounce), **Is Trigger** (physics system ignores the collider for collision response but still reports triggers), **Used By Effector** (allows an attached Effector 2D to use this collider), **Offset** (local offset of geometry) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/box-collider-2d-reference.html
- **[api]** Unity 6 replaces the old boolean **Used By Composite** with a **Composite Operation** enum: **None**, **Merge**, **Intersect**, **Difference**, **Flip** — controls how this collider combines with a Composite Collider 2D on the same GameObject — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/box-collider-2d-reference.html
- **[gotcha]** When **Composite Operation** is anything other than None, the Composite Collider 2D controls Material, Is Trigger, Used By Effector, and Edge Radius — the child collider's own values are ignored — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/box-collider-2d-reference.html
- **[concept]** **Auto Tiling** (Box/other): when the Sprite Renderer Draw Mode = **Tiled**, enabling this auto-updates the collider shape when sprite dimensions change; without it the collider geometry does not repeat — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/box-collider-2d-reference.html
- **[concept]** **Edge Radius** (Box/Edge/Capsule etc.): forms a radius around the collider edge giving rounded convex corners. Default **0** — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/box-collider-2d-reference.html
- **[concept]** Box-specific fields: **Size** (box size in local units), **Offset**. Capsule adds **Direction** (Vertical/Horizontal). Circle uses **Radius**. Polygon uses editable **Points**. Edge is an open path (also supports Edge Radius and adjacent start/end points) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/box-collider-2d-reference.html
- **[concept]** **Layer Overrides** on colliders: Layer Override Priority, Include Layers, Exclude Layers, Force Send Layers, Force Receive Layers, Contact Capture Layers, Callback Layers — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/box-collider-2d-reference.html

## Composite Collider 2D
- **[concept]** Composite Collider 2D has no inherent shape — it merges the geometry of Box, Circle, Capsule, and Polygon Collider 2Ds set to feed into it (via their Composite Operation). It requires a Rigidbody 2D on the same GameObject — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/composite-collider/composite-collider-2d-reference.html
- **[concept]** **Geometry Type** enum: **Outlines** (hollow, edge-based collisions like Edge Collider 2D) and **Polygons** (solid filled polygons like Polygon Collider 2D) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/composite-collider/composite-collider-2d-reference.html
- **[concept]** **Generation Type** enum: **Synchronous** (regenerates geometry immediately on any change) and **Manual** (must call `CompositeCollider2D.GenerateGeometry()` or press the Inspector button) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/composite-collider/composite-collider-2d-reference.html
- **[concept]** **Use Delaunay Mesh**: enables Delaunay triangulation for complex shapes — more accurate but lower performance — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/composite-collider/composite-collider-2d-reference.html
- **[concept]** **Vertex Distance**: minimum spacing between vertices (closer ones are removed). **Offset Distance**: threshold for merging vertices between shapes (keep ≤ ~1% of sprite length) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/composite-collider/composite-collider-2d-reference.html
- **[concept]** Also exposes Material, Is Trigger, Used By Effector, Offset, and full Layer Override set (these override child colliders that feed into it) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/collider/composite-collider/composite-collider-2d-reference.html

## Physics Material 2D
- **[workflow]** Create via **Assets > Create > 2D > Physics Material 2D**; assign to the Material property of a Collider 2D or Rigidbody 2D — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/physics-material-2d-reference.html
- **[concept]** Fields: **Friction** (surface friction coefficient) and **Bounciness** (0 = no bounce, 1 = perfect bounce) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/physics-material-2d-reference.html
- **[default]** **Friction Combine** default = **Mean**; **Bounciness Combine** default = **Maximum** (note the asymmetry). Combine modes: Average, Mean, Multiply, Minimum, Maximum (`PhysicsMaterialCombine2D`) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/physics-material-2d-reference.html
- **[gotcha]** When two colliders have different combine modes, the highest-priority mode wins in order Maximum > Minimum > Multiply > Mean/Average — src: https://docs.unity3d.com/6000.0/Documentation/ScriptReference/PhysicsMaterialCombine2D.html

## Physics 2D Settings / Layer Collision Matrix
- **[concept]** The **Layer Collision Matrix** (Project Settings > Physics 2D) controls whether colliders on different layers can contact. A checked cell = contact allowed; cleared = never allowed — src: https://docs.unity3d.com/6000.2/Documentation/Manual/class-Physics2DSettings.html
- **[perf]** Enable only the minimum set of layer contacts you need — clearing unnecessary layer pairs reduces broadphase work and improves performance — src: https://docs.unity3d.com/6000.2/Documentation/Manual/class-Physics2DSettings.html

## Effectors 2D
- **[concept]** Effector types: **Area Effector 2D** (vary force + angle magnitude over an area), **Buoyancy Effector 2D** (buoyancy, fluid flow, fluid drag), **Point Effector 2D** (attract/repel toward/from a source point), **Platform Effector 2D** (one-way collisions and platform behavior), **Surface Effector 2D** (conveyor belt tangent forces) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/effectors/effectors-2d-landing.html
- **[gotcha]** Every effector requires the collider it acts on to have **Used By Effector** enabled — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/effectors/effectors-2d-landing.html
- **[concept]** Shared effector fields: **Use Collider Mask** (if off, the global collision matrix is used) and **Collider Mask** (which layers the effector affects) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/effectors/platform-effector-2d-reference.html
- **[concept]** Platform Effector 2D fields: Use Collider Mask, Collider Mask, **Use One Way**, **Use One Way Grouping**, **Surface Arc**, **Use Side Friction**, **Use Side Bounce**, **Side Arc** — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/effectors/platform-effector-2d-reference.html
- **[gotcha]** **Use One Way Grouping** makes all one-way-disabled contacts act on all colliders together — needed when multiple colliders pass through a platform as a group — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/effectors/platform-effector-2d-reference.html
- **[concept]** Surface Effector 2D: **Speed**, **Speed Variation**, **Force Scale** (conveyor-belt tangent force). Buoyancy Effector 2D: **Surface Level**, **Density**, Linear/Angular Drag, Flow Angle/Magnitude — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/effectors/surface-effector-2d-reference.html

## Joints 2D
- **[concept]** 2D joint types: **Distance** (maintain fixed distance), **Fixed** (hold two bodies together; spring-implemented so small motion possible), **Friction** (constrain relative motion via friction), **Hinge** (rotate about a point — doors/chains/pendulums), **Relative** (maintain relative position/offset), **Slider** (slide along one axis/line), **Spring** (spring force keeping bodies a distance apart), **Target** (attach to a target position, spring-like), **Wheel** (suspension spring + hinge for rolling wheels) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/joints/2d-joints-landing.html
- **[concept]** Wheel Joint 2D ≈ Slider Joint 2D (no motor/limit) combined with Hinge Joint 2D (no limit) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/joints/wheel-joint-2d-fundamentals.html
- **[concept]** Shared joint concepts: **Connected Rigidbody**, **Anchor**, **Connected Anchor**, **Break Force**, **Break Torque**, **Enable Collision** (between the connected bodies) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/2d-physics/joints/2d-joints-landing.html

## URP 2D Renderer setup (Renderer 2D Data asset)
- **[workflow]** Create a URP Asset with 2D Renderer via right-click **Create > Rendering > URP Asset (with 2D Renderer)**, then assign it in the Graphics/Quality settings. The 2D lighting system ships its own Light 2D components, Shader Graph sub-targets, and a custom 2D render pass — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/Lights-2D-intro.html
- **[concept]** **Default Material Type** on the Renderer 2D asset: **Lit** (uses 2D/Sprite-Lit-Default — affected by 2D lights), **Unlit** (2D/Sprite-Unlit-Default — ignores 2D lights), **Custom** (user material) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/2DRendererData-overview.html
- **[concept]** **Blend Styles**: each Blend Style on the Renderer 2D asset maps to a Blend Style option on Light 2D. Blend Mode: **Additive** (adds light color to background), **Multiply**, **Subtractive**. **Mask Texture Channel**: Red/Green/Blue/Alpha (+ inverted variants) selects which mask channel gates the light — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/2DRendererData-overview.html
- **[perf]** **Light Render Texture Scale** default **0.5** (renders lights at 50% resolution). **Max Light Render Textures** default **4**. HDR Emulation Scale multiplies high-intensity lights on non-HDR platforms — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/2DRendererData-overview.html
- **[perf]** Fewer Blend Styles = better performance — reducing the number of blend styles used directly improves rendering perf — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/Lights-2D-intro.html
- **[concept]** **Transparency Sort Mode/Axis**: Default / Perspective / Orthographic / Custom Axis — controls 2D depth sorting — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/2DRendererData-overview.html

## Light 2D
- **[concept]** Light Types: **Freeform** (custom editable polygon shape), **Sprite** (light in the shape of a sprite), **Spot** (point source, cone-shaped area — was called "Point"), **Global** (lights all 2D GameObjects equally, no attenuation). Create via **GameObject > Light**. **Parametric** is deprecated — use the Render Pipeline Converter "Parametric To Freeform Light Upgrade" to migrate — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/2DLightProperties.html
- **[default]** Shared fields: **Light Type**, **Color**, **Intensity** (default **1**), **Blend Style** (Multiply / Additive / Multiply with Mask (R) / Additive with Mask (R)), **Light Order** (render order; supports negatives), **Overlap Operation** (**Additive** default, or **Alpha Blend**), **Target Sorting Layers** (which sorting layers this light illuminates) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/2DLightProperties.html
- **[concept]** Spot light fields: **Inner/Outer Radius**, **Inner/Outer Spot Angle**, **Falloff Strength**. Freeform light: **Falloff** (extends falloff reach), **Falloff Strength** (edge softness), plus scene-view **Edit Shape** — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/2DLightProperties.html
- **[concept]** **Shadows** (non-Global lights): **Strength** (0–1, how much light is blocked), **Softness**, **Falloff Strength** — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/2DLightProperties.html
- **[concept]** **Normal Map** on lights (all except Global): **Normal Map Quality** = Disabled / **Fast** / **Accurate**; when enabled, **Distance** (light-to-sprite distance) appears — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/2DLightProperties.html
- **[gotcha]** Global lights cannot use normal maps and cannot cast shadows — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/SecondaryTextures.html

## Shadow Caster 2D
- **[concept]** **Casting Source** enum: **Sprite Skin** (bone bounding box), **Sprite Renderer** (Sprite Editor shapes), **Collider 2D** (attached collider shape), **Shape Editor** (custom-edited outline), **None** — defines where the shadow shape comes from — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/ShadowCaster2D.html
- **[concept]** **Casting Option** enum: **Self Shadow**, **Cast Shadow** (default), **Cast and Self Shadow**, **None** — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/ShadowCaster2D.html
- **[default]** **Target Sorting Layers** default **Everything** (which sorting layers receive shadows from this caster). **Trim Edge** default **0.2** (Sprite Renderer source only). **Alpha Cutoff** default **0.1** (minimum alpha to cast a shadow; Sprite Renderer source only) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/ShadowCaster2D.html
- **[gotcha]** Shadows only appear if the affecting Light 2D has Shadows enabled (Strength > 0) and is not a Global light — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/2DShadows.html

## Normal Maps / Secondary Textures (2D lighting)
- **[workflow]** Normal maps and mask maps are added as **secondary textures** in the Sprite Editor's **Secondary Textures** overlay; select from the dropdown with the exact reserved names **_NormalMap** and **_MaskTex** — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/SecondaryTextures.html
- **[gotcha]** Import normal maps as Texture Type "Normal Map"; mask maps as "Default". The Sprite Renderer must use a compatible lit material (Sprite-Lit-Default) — standard sprite materials won't work — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/SecondaryTextures.html
- **[workflow]** To activate: set the light's **Use Normal Map** to Fast/Accurate; for masks set the light's Blend Style to "Multiply with Mask (R)" or "Additive with Mask (R)" — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/SecondaryTextures.html
- **[perf]** Adding a normal map inserts a depth pre-pass per layer batch rendering a full-size render texture — can reduce performance — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/SecondaryTextures.html

## Sprite Shape
- **[concept]** A **Sprite Shape Profile** asset is created from the Editor main menu as **Open Shape** or **Closed Shape**. Open Shape = single-edge outline with tiled sprites (platforms/level edges), auto-enables **Open Ended**. Closed Shape = enclosed area with an optional tiled **Fill** texture, **Open Ended** disabled — src: https://docs.unity3d.com/Packages/com.unity.2d.spriteshape@10.0/manual/SSProfile.html
- **[concept]** **Sprite Shape Controller** is auto-added when a Profile is dragged into the scene; you edit the outline through its control points — src: https://docs.unity3d.com/Packages/com.unity.2d.spriteshape@10.0/manual/SSController.html
- **[concept]** Controller Spline fields: **Profile**, **Edit Spline**, **Detail** (High/Medium/Low tessellation), **Open Ended**, **Adaptive UV** (default on — seamlessly tiles sprites by deforming between control points), **Enable Tangents** (needed for some URP shader effects), **Corner Threshold** (default 30° — angle at which a point counts as a corner) — src: https://docs.unity3d.com/Packages/com.unity.2d.spriteshape@10.0/manual/SSController.html
- **[default]** Fill fields: **Fill Tessellation (C# Job)**, **Stretch UV**, **Pixels Per Unit** (default **100**; higher = smaller texture), **World Space UV** — src: https://docs.unity3d.com/Packages/com.unity.2d.spriteshape@10.0/manual/SSController.html
- **[concept]** Collider fields: **Update Collider** (default on), **Offset** (range -0.5..0.5, default 0), **Detail** (High/Medium/Low), **Collider Type** (Polygon Collider 2D or Edge Collider 2D) — colliders auto-remove colinear control points — src: https://docs.unity3d.com/Packages/com.unity.2d.spriteshape@10.0/manual/SSController.html
- **[gotcha]** Disabling Adaptive UV can cut off sprites if control-point spacing is smaller than the sprite width; stretched corners need matching Height values and identical sprite pivots — src: https://docs.unity3d.com/Packages/com.unity.2d.spriteshape@10.0/manual/SSController.html

## 2D Animation — Sprite Skin & Skinning Editor
- **[concept]** The **Skinning Editor** (Sprite Editor module) creates the bone skeleton, geometry mesh, and skin weights. Paint weights to control how much each bone influences mesh vertices / how the mesh deforms — src: https://docs.unity3d.com/Packages/com.unity.2d.animation@8.0/manual/CharacterRig.html
- **[concept]** **Sprite Skin** deforms a sprite using GameObject Transforms that represent the rigged+weighted bones from the Skinning Editor. The PSD Importer auto-adds Sprite Skin to sprites that contain bone/weight data — src: https://docs.unity3d.com/Packages/com.unity.2d.animation@6.0/manual/SpriteSkin.html
- **[concept]** Sprite Skin fields: **Always Update** (deform even when off-camera), **Auto Rebind** (auto-locate Transforms matching bone names), **Root Bone**, **Bones** (list), **Create Bones** (button — generates Transforms for rigged bones), **Reset Bind Pose**, **Enable Batching** — src: https://docs.unity3d.com/Packages/com.unity.2d.animation@6.0/manual/SpriteSkin.html
- **[gotcha]** **Auto Rebind** requires GameObject Transform names + hierarchy to exactly match the Skinning Editor bones — renaming a bone requires renaming its Transform — src: https://docs.unity3d.com/Packages/com.unity.2d.animation@6.0/manual/SpriteSkin.html
- **[perf]** **Enable Batching** uses Burst compilation and only appears when both Burst (1.4.1+) and Collections (0.9.0-preview.6+) packages are installed — src: https://docs.unity3d.com/Packages/com.unity.2d.animation@6.0/manual/SpriteSkin.html

## PSD Importer
- **[concept]** The PSD Importer imports `.psb` files and can generate a Prefab with sprites laid out per source-file layer, auto-attaching Sprite Skin components where bone/weight data exists. A **.skeleton** asset holds a bone hierarchy that can be assigned to **Main Skeleton** to auto-rig the generated character prefab — src: https://docs.unity3d.com/Packages/com.unity.2d.psdimporter@11.0/manual/PSD-importer-properties.html
- **[default]** Key inspector fields: **Texture Type** = Sprite (2D and UI); **Sprite Mode** (Single / Multiple, default Multiple); **Import Mode** (Individual Sprites (Mosaic) / Merged); **Mesh Type** (Full Rect / Tight, default Tight); **Pixels Per Unit**; **Extrude Edges**; **Pivot**; **Use Layer Group** (needs Character Rig); **Include Hidden Layers**; **Keep Duplicate Name**; **Use as Rig (Character Rig)**; **Main Skeleton** — src: https://docs.unity3d.com/Packages/com.unity.2d.psdimporter@11.0/manual/PSD-importer-properties.html
- **[gotcha]** Duplicate layer names break the "Use Layer Name" mapping (unique names required). Sprites under 32×32 px are forced to Full Rect mesh regardless of selection. Layer Group only available with Character Rig enabled + Individual Sprites mode — src: https://docs.unity3d.com/Packages/com.unity.2d.psdimporter@11.0/manual/PSD-importer-properties.html

## Pixel Perfect Camera (URP)
- **[gotcha]** In URP the Pixel Perfect Camera is built into the pipeline — do NOT install the separate `com.unity.2d.pixel-perfect` package (that one only targets the Built-In Render Pipeline) — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/2d-pixelperfect-ref.html
- **[concept]** Fields: **Asset Pixels Per Unit** (sprite pixels per world unit), **Reference Resolution** (X/Y original design resolution, e.g. 320×180), **Filter Mode** (Point / **Retro AA**), **Crop Frame** (None / Pillarbox / Letterbox / Windowbox / Stretch Fill), **Grid Snapping** (None / Pixel Snapping / Upscale Render Texture), plus read-only **Current Pixel Ratio** — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/2d-pixelperfect-ref.html
- **[gotcha]** All sprites in the scene must share the same Pixels Per Unit and it must match the camera's Asset Pixels Per Unit — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/2d-pixelperfect-ref.html
- **[concept]** **Upscale Render Texture** (Grid Snapping) renders to a temp texture at the Reference Resolution then upscales — yields unaliased, unrotated pixels. **Retro AA** (Filter Mode) upscales to nearest integer multiple of the reference resolution then bilinear-upscales to the screen — src: https://docs.unity3d.com/6000.0/Documentation/Manual/urp/2d-pixelperfect-ref.html
